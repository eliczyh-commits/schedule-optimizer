# 项目说明：旬度内贸排产优化模型

## 业务目标

本项目用于建筑用钢内贸旬度排产。生产部门给出每旬总资源，外贸排产固定占用产线，内贸资源为总资源扣减外贸资源。模型在满足保供、产线天数、规格比例和抗震占比约束下，为内贸生成排产方案。

最终目标：

- HRB500E 和 600兆帕按需求表必须排满。
- 400兆帕钢筋在约束内最大化总现金流。
- 400兆帕客户需求预报只生成建议，不反向调整原方案。

## 关键业务规则

- 外贸排产是固定输入，只计算占用天数，不参与优化。
- 如果外贸占用天数已经超过某条产线本旬生产天数，必须直接报错，不继续生成内贸方案。
- 吨位按 100 吨取整。
- HRB500E 和 600兆帕需求必须 100% 满足；如果排不下，直接报错。
- 600兆帕包含 `T63E/E/G` 和 `T63/E/G`。
- `T63E/E/G` 可排规格：12、14、16、18、20、22、25、28、32。
- `T63/E/G` 只排规格：10。
- 400兆帕钢筋包含 `HRB400` 和 `HRB400E`。
- HRB400E 在 400兆帕钢筋中占比至少 65%。
- 400兆帕各规格按比例上下限控制。
- 某个产线、牌号、规格只有在 `产品生产效率表` 存在日产量时才允许排产。

## 输入数据

Excel/网页录入需要覆盖这些表：

- `旬度资源说明`：旬度、总资源、外贸、内贸、500兆帕保供量、600兆帕保供量、HRB400/HRB400E资源量。
- `保供量`：旬度、牌号、规格、吨位，适用于 HRB500E、T63E/E/G、T63/E/G。
- `生产日历`：产线、生产天数。
- `外贸排产明细`：旬度、产线、牌号、规格、吨位。
- `产品生产效率表`：产线、牌号、规格、日产量。
- `现金流量表`：牌号、规格、现金流。
- `规格比例约束`：400兆帕规格、比例下限、比例上限。
- `400兆帕客户需求预报`：旬度、牌号、规格、客户预报吨位。

资源表中小于 1000 的资源量按“万吨”理解，例如 0.6 表示 6000 吨；大于等于 1000 的数按吨理解。

## 输出数据

`内贸排产方案`：

- 旬度
- 牌号
- 规格
- 吨位
- 生产条线

`方案校验`：

- 产线生产天数是否超限
- HRB500E需求是否排满
- 600兆帕需求是否排满
- 400兆帕资源是否排满
- HRB400E抗震占比是否达标
- 400兆帕各规格比例是否达标
- 400兆帕总现金流

`400兆帕需求建议`：

- 旬度
- 牌号
- 规格
- 客户预报吨位
- 排产方案吨位
- 折扣
- 简要原因

折扣 = 排产方案吨位 / 客户预报吨位。建议只解释差异，不改变排产方案。

## 项目文件

- `schedule_model.py`：Excel 版主模型和优化内核。
- `web_app.py`：本地网页应用服务。
- `web/index.html`、`web/styles.css`、`web/app.js`：网页录入与结果界面。
- `requirements.txt`：依赖，主要是 `openpyxl` 和 `pulp`。

## 常用命令

安装依赖：

```powershell
py -m pip install -r requirements.txt --target .python-libs2
```

如果点号目录权限异常，使用：

```powershell
py -m pip install -r requirements.txt --target libs
```

运行 Excel 版：

```powershell
py schedule_model.py --input "C:\Users\24335\Desktop\排产所需文件 - 副本.xlsx" --output "C:\Users\24335\Documents\旬度排产模型\排产结果.xlsx"
```

运行网页应用：

```powershell
py web_app.py
```

打开：

```text
http://127.0.0.1:8765
```

## 当前注意事项

- 如果出现“外贸已占用 X 天，超过生产天数 Y 天”，这是硬性业务报错，应先调整外贸排产、生产日历或对应日产量。
- 不要在外贸超产线天数时继续尝试优化内贸。
- 不要把 400兆帕客户预报量作为硬约束；它只用于建议折扣。
